# Code Generation Component

The `CodeGeneration` component is designed to showcase code generation workflows in documentation. It provides a clean, structured way to display the relationship between input source code and generated output code.

## Features

- **Flexible Content Sources**: Support for both remote URLs and inline content
- **Syntax Highlighting**: Full Prism.js language support for both source and output
- **Collapsible Sections**: Optional collapsible code blocks for better readability
- **Source Links**: Automatic generation of "View Source" links for remote content
- **Responsive Design**: Adapts to different screen sizes
- **Dark Mode Support**: Automatic theme switching
- **GitHub Integration**: Automatic URL transformation and line linking for GitHub URLs

## Component Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `command` | string | - | Description of the command or generation process |
| `sourceSummary` | string | - | Summary/title for the source code section |
| `sourceUrl` | string | - | URL to fetch source code from (GitHub URLs supported) |
| `sourceContent` | string | - | Direct source code content |
| `sourceLanguage` | string | `'text'` | Language for source code syntax highlighting |
| `sourceVisibleRange` | string | - | Range of lines to show (e.g., "1,20") |
| `outputSummary` | string | - | Summary/title for the output code section |
| `outputUrl` | string | - | URL to fetch output code from |
| `outputContent` | string | - | Direct output code content |
| `outputLanguage` | string | `'text'` | Language for output code syntax highlighting |
| `outputVisibleRange` | string | - | Range of lines to show from output |
| `collapsed` | boolean | `false` | Whether code sections should be collapsed by default |
| `showLegend` | boolean | `true` | Whether to show "View Source" links |
| `afterText` | string | - | Additional descriptive text to show after the generation example |

## Supported Languages

The component supports all languages available in Prism.js, including:

- `javascript`, `typescript`, `jsx`, `tsx`
- `java`, `kotlin`, `scala`
- `python`, `ruby`, `php`
- `yaml`, `json`, `xml`
- `zdl` (ZenWave Domain Language)
- `sql`, `bash`, `dockerfile`
- `markdown`, `text`

## Legacy Support

```html
<div className="codegeneration">
  <div className="codegeneration__command">Command description</div>
  <div className="codegeneration__source">
    <RemoteCode url="..." language="..." />
  </div>
  <div className="codegeneration__output">
    <RemoteCode url="..." language="..." />
  </div>
</div>
```

## Best Practices

1. **Use descriptive summaries**: Help users understand what each code section represents
2. **Choose appropriate languages**: Use specific language identifiers for better syntax highlighting
3. **Consider collapsing long code**: Use `collapsed={true}` for lengthy code examples
4. **Provide context**: Use the `command` prop to explain the generation process
5. **Use visible ranges**: For large files, show only relevant sections using `visibleRange`
6. **Enable source links**: Keep `showLegend={true}` to help users explore the full source

## GitHub Integration

The component automatically handles GitHub URLs:

- Transforms blob URLs to raw URLs for fetching
- Generates proper source links with line numbers
- Supports line ranges in fragment links (e.g., `#L10-L20`)

Example:
```jsx
<CodeGeneration
  sourceUrl="https://github.com/user/repo/blob/main/file.java"
  sourceVisibleRange="10,30"
  // Automatically creates link: https://github.com/user/repo/blob/main/file.java#L10-L30
/>
```
