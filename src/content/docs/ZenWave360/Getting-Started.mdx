# Getting Started with ZenWave 360

## Installation

<div class="getting-started-intro">

We recommend using [SDKMAN!](https://sdkman.io/) to install ZenWave SDK requirements: [Java 17+](https://sdkman.io/jdks), [Maven](https://sdkman.io/sdks#maven), and [JBang](https://sdkman.io/sdks#jbang).

NOTE: Temurin (Eclipse) is a good option for Java JDK because it is high performance, enterprise-caliber, and open-source licensed.

</div>

## ZenWave SDK Installation

Use JBang to install an _evergreen_ self-updating CLI for the ZenWave SDK.

```bash
jbang alias add --force --fresh --name=zw release@zenwave360/zenwave-sdk
```

**NOTE:** You can install ZenWave versions from different channels: `release` for the latest stable version and `snapshots` for development versions. They can be installed side by side using different `--name=<alias>` values, or independently.

## Install ZenWave Editor IntelliJ Plugin

<div class="getting-started-section getting-started-section--plugin">

## Spring Boot Base Project

ZenWave SDK only generates business logic code and tests. It is expected to work with any compatible Spring Boot project. The default backend generator is based on a Maven single-module project.

We expect most companies and teams to have their own starter base project. If you do not have one, you can create one from https://start.spring.io or clone one of the ZenWave 360 base projects from our GitHub playground repository.

<div class="getting-started-starters">

> NOTE: These baseline projects come with OpenAPI Generator and ZenWave SDK Maven plugins already configured, so you do not have to.

ZenWave SDK generated code was tested with Spring Boot up to 3.5.0, Hibernate 6.x, MongoDB 5, and Spring Cloud 2024.0.0. Because it only generates business code without any exotic dependency, it should work with any compatible version.

Once you have installed ZenWave SDK CLI and IntelliJ Plugin, you are ready to start modeling your domain and generating code.

## ZDL Modeling and Code Generation

Scroll left/right for a quick overview of how ZDL modeling looks with an example. Then follow the instructions below.

<ImageCarousel ariaLabel="ZDL modeling and code generation screenshots" slides={zdlModelingSlides} />

Create a new ZDL model or use one of the ZDL examples as a base:

<div class="getting-started-steps">

1. Create a new file with the `.zdl` extension.
2. Model your bounded context by adding entities, services, inputs, and events.
3. Add ZenWave SDK plugins to the ZDL `config` section. You can use IntelliJ live templates for this task.
4. Follow these steps to generate code and tests from your model:
    1. Generate API definitions from your model. OpenAPI and AsyncAPI v2/v3 are supported.
    2. Customize your API definitions. They are the source of truth for outbound and inbound communications.
    3. Configure API-first generators in `pom.xml` for OpenAPI and AsyncAPI.
    4. Generate your core backend code and tests. ZenWave SDK can generate CRUD implementation for your services.
    5. Customize generated code by implementing non-CRUD service commands, and customize generated tests.
    6. Implement adapter code for web and events, connecting API-first generated code with your core backend. ZenWave SDK can generate stub implementations for some adapters.
5. Run your tests and start your application.

</div>
